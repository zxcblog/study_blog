# 使用docker-compose搭建集群

etcd通常都会以集群方式部署，以避免单点故障。etcd构建自身高可用集群主要有三种形式
- 静态配置
  - 预先知道etcd集群中有哪些节点，在启动时通过 --initial-cluster参数直接指定好etcd的各个节点地址
- etcd动态发现
  - 静态配置前提是在搭建集群之前已经提前知道各节点的信息，而实际应用中可能存在预先并不知道个节点ip的情
 况，这时可通过已搭建的etcd来辅助搭建新的etcd集群。通过已有的etcd集群作为数据交互点，然后再扩展新的集
 群时，实现通过已有集群进行服务发现的机制。比如官方提供的discovery.etcd.io
- DNS动态发现
  - 通过DNS查询方法获取其他节点地址信息



# 静态配置文件讲解
文件地址 `services/etcd/conf/etcd1.yaml`
```yaml

# etcd节点名字
name: etcd1
# etcd数据存储位置
data-dir: /etcd/data

# 监听客户端请求的地址列表， 格式 http://localhost:2379 多个用逗号分隔
listen-client-urls: http://172.26.0.101:2379, http://127.0.0.1:2379

# 如果 listen-client-urls配置了多个监听客户端请求的地址，这个参数可以给出建议客户端使用什么地址访问etcd
advertise-client-urls: http://172.26.0.101:2379

# 服务端节点之间通讯的监听地址， 格式 http://localhost:2380
listen-peer-urls: http://172.26.0.101:2380

# 建议服务端之间通讯使用的地址列表
initial-advertise-peer-urls: http://172.26.0.101:2380

# etcd启动的时候，通过这个配置找到其他etcd节点的地址列表，格式： 节点名字=http://节点ip:2380,...
initial-cluster: etcd1=http://172.26.0.101:2380,etcd2=http://172.26.0.102:2380,etcd3=http://172.26.0.103:2380

initial-cluster-token: etcd-cluster-token

# 初始化的时候， 集群的状态 new 或者existing两种状态， new 新建的集群， exists表示加入已存在的集群
initial-cluster-state: new
```

# etcd动态发现
使用已有的etcd集群来注册和启动

## 实现机制
discovery services protocol 可以帮助新的etcd成员使用共享url在集群引导阶段发现所有其他成员。该协议使用新的发现令牌
来引导一个唯一的etcd集群，一个发现令牌只能代表一个etcd集群，只要此令牌上的发现协议启动，即使它中途失败，也不能用于引导另一个etcd集群。

## 协议的工作流程
discovery protocol 是使用内部etcd集群来协调新集群的引导程序。首先，所有新成员都与发现服务交互，并帮助生成预期的成员列表，然后，每个成员
使用比列表引导服务器，该列表执行与--initial-cluster标志相同的功能，即设置所有集群的成员信息

> 注意： discovery services protocol 仅用于集群引导阶段，不能用于运行时重新配置或集群监视

## 创建集群
```shell
# 创建集群 size为集群节点数量，未指定则为3
$ curl https://discovery.etcd.io/new?size=3
# 此处会返回节点令牌
```

创建yaml文件
```yaml
# etcd节点名字
name: etcd1
# etcd数据存储位置
data-dir: /etcd/data

# 监听客户端请求的地址列表， 格式 http://localhost:2379 多个用逗号分隔
listen-client-urls: http://172.26.0.101:2379, http://127.0.0.1:2379

# 如果 listen-client-urls配置了多个监听客户端请求的地址，这个参数可以给出建议客户端使用什么地址访问etcd
advertise-client-urls: http://172.26.0.101:2379

# 服务端节点之间通讯的监听地址， 格式 http://localhost:2380
listen-peer-urls: http://172.26.0.101:2380

# 建议服务端之间通讯使用的地址列表
initial-advertise-peer-urls: http://172.26.0.101:2380

# 指定节点令牌
discovery: https://discovery.etcd.io/节点令牌
```
